## Key Technical Findings

### Qualcomm Module Location Anomaly

The most important insight from this session: Qualcomm's out-of-tree kernel modules live in `vendor/qcom/opensource/` — a path prefix that the L2 router associates with HAL implementations. This creates a systematic routing failure for QC kernel issues that will need to be addressed in Phase 6.4 (L3-aware routing upgrade).

The current routing behavior (L2 → hal-vendor-interface-expert → L3 escalation) is documented in TC-101–TC-105 notes.

### SoC Codename Coverage

Documented SoC → GKI branch mapping from SM8350 (lahaina, 2021) through SM8750 (sun, 2025). This table is critical for BSP engineers who need to identify the correct GKI kernel branch for a given device.

### ADSP/PIL Lifecycle

The PIL/remoteproc boot sequence for ADSP (critical for audio debugging) is not documented in any L2 skill. It requires QC-specific knowledge: `/vendor/firmware/adsp.mbn` loading, FastRPC session management, APR transport recovery after crash.
