## Key Findings

### 1. AVF/pKVM Major Expansion (HS-037)

Android 16 significantly expands AVF beyond the A15 feature set:
- **AVF LL-NDK**: Vendors can launch VMs from vendor partition using Google-managed AVF
- **Early boot VMs**: VMs run before full Android init; KeyMint HAL in pVM is a key use case
- **FF-A support**: pKVM now uses standardized Firmware Framework for Arm communication with TrustZone (replaces ad-hoc SMC calls)
- **Ferrochrome**: Debian-based Linux terminal in a VM (developer feature)
- **Microdroid**: Resizable encrypted storage + 16K protected VM support
- **Trusty OS in pVMs**: TrustZone-style TAs can run in protected VMs
- **Device assignment**: Graduated from experimental — direct hardware access for pVMs
- **Hypervisor tracing**: Structured logging for pKVM debugging

This is the largest single-version expansion of AVF since its introduction.

### 2. SELinux IOCTL Hardening + KeyMint 4.0 (HS-038)

- **QPR2 SELinux macro**: Blocks restricted IOCTLs in production builds, limits profiling IOCTLs to shell/debuggable. Primary target: GPU drivers.
- **KeyMint 4.0**: New `moduleHash` field in attestation certs for APEX module integrity verification.

### 3. Connectivity: Unified Ranging + AIS + IMS (HS-039)

- **Ranging module**: Unifies UWB, BT Channel Sounding, BT RSSI, and WiFi RTT under one API
- **AIS (Android Information Service)**: Bluetooth GATT characteristic exposing Android API level to peripherals
- **IMS expansion**: Traffic session management, EPS fallback triggers, emergency callback listener

### 4. GBL Detail Update (supplements HS-034)

New details from GBL overview page confirm 5 core components:
1. Core Android Boot Logic
2. Fastboot
3. Vendor Extensions
4. UEFI Protocol Handlers
5. Android-Specific UEFI Protocols (AVB, Fastboot, slot selection, OS config)

Requirement: ARM-64 devices shipping with Android 16 **strongly recommended** to deploy Google-certified GBL.

### 5. AOSP Publication Model Change

Starting 2026, AOSP source code published on Q2/Q4 cadence (not continuous).
Recommendation: use `android-latest-release` branch instead of `aosp-main`.

### 6. Other A16 Items (already partially covered)

- **16KB page size**: TLS memory optimization (~8KB savings per thread via isolated basename/dirname buffers). Google Play requires 16KB support for apps targeting A15+ (Nov 2025 deadline passed).
- **APV codec, CAP AIDL fix, HDR enhancements**: Covered in HS-035.
- **GKI 6.12**: Covered in HS-033.
