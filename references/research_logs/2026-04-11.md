## Key Findings

### 1. AOSP Biannual Source Code Cadence (Confirmed)
- Effective 2026, source code published to AOSP in Q2 and Q4 only
- April 2026 expected for Android 16 source push
- Recommendation: use `android-latest-release` branch instead of `aosp-main`
- **Impact for skill set**: When A16 source drops in April, all dirty skills (6 of 13) can be validated against actual AOSP tree
- **Action**: Noted in research log; no hindsight note needed (operational cadence, not architectural)

### 2. Bluetooth Bond Loss Handling — New Intents (A16)
- `ACTION_KEY_MISSING`: broadcast when remote bond loss is detected
- `ACTION_ENCRYPTION_CHANGE`: broadcast when link encryption status/algorithm/key size changes
- Part of improved Bluetooth bond loss handling in Android 16
- **Path**: `packages/modules/Bluetooth/`
- **Action**: Updated HS-039 with new details

### 3. Bluetooth LE Audio Sharing (A16)
- Bluetooth Audio Sharing allows connecting multiple LE Audio headphones simultaneously
- Requires Bluetooth LE-supported earbuds
- Framework-level multi-device audio routing change
- **Path**: `packages/modules/Bluetooth/`, `frameworks/av/`
- **Action**: Updated HS-039 with new details

### 4. CompanionDeviceManager removeBond API (A16)
- New public `removeBond(int)` API in `CompanionDeviceManager`
- Apps targeting A16 can unpair Bluetooth devices programmatically
- **Path**: `frameworks/base/core/`
- **Action**: Noted; minor API addition, included in HS-039 update

### 5. IOCTL Hardening — SoC-Specific Detail
- IOCTL allowlists must be synchronized with both userspace and kernel drivers
- Individual SoCs must update IOCTL lists per device upon driver release
- ARM publishes updated IOCTL lists for Mali GPU upon driver updates
- Tested on Pixel 6-9 (Mali GPU) with no measurable GPU performance impact
- Change is opt-in for OEMs but will ship as default in a future AOSP release
- **Action**: Updated HS-038 with SoC-specific operational detail

### 6. GKI 6.12 Errata — No New Items
- Bounds sanitizer `__counted_by` issue and CONFIG_OF_DYNAMIC regression already captured in HS-033
- CONFIG_UBSAN_SIGNED_WRAP disabled, CONFIG_UBSAN_TRAP temporarily disabled during Clang update — already noted
- No new errata entries since last session
- Latest release remains android16-6.12-2026-03 (kernel 6.12.69)

### 7. AVF Mandatory in Android 16 (Confirmed)
- AVF is mandatory starting from Android 16 (previously optional)
- Already captured in HS-037
- Linux terminal (Ferrochrome) feature expanded from A15 QPR2 beta
