## Key Findings

### 1. GBL Source Path Confirmed (bootable/libbootloader)
- **Previous state**: HS-034 marked GBL source location as "TBD — not yet in public AOSP tree"
- **Update**: GBL source is at `bootable/libbootloader` in the `uefi-gbl-mainline` manifest branch
- Build uses Bazel: `tools/bazel run //bootable/libbootloader:gbl_efi_dist`
- Partition layout: `android_esp_a`/`android_esp_b` (FAT, ≥8 MB)
- EFI path: `/EFI/BOOT/BOOTAA64.EFI`
- UEFI protocols: `GBL_EFI_AVB_PROTOCOL`, `GBL_EFI_BOOT_CONTROL_PROTOCOL`, `GBL_EFI_AVF_PROTOCOL`
- Firmware API level: `gbl_fw_api_level` UEFI variable
- **Action**: Updated HS-034 with confirmed details

### 2. Qualcomm GBL Exploit (March 2026)
- Three-vulnerability chain targeting Qualcomm's GBL Library implementation
- Could unlock bootloaders on Android 16 flagships
- Qualcomm patched in early March 2026
- Highlights new attack surface from GBL standardization
- **Action**: Created HS-040

### 3. AIDL CAP Interface Named (AudioHalCapConfiguration.aidl)
- Specific new AIDL definition: `AudioHalCapConfiguration.aidl`
- Framework now loads CAP config via AIDL HAL directly, replacing XML parsing
- Cuttlefish Auto migrated to AIDL CAP as reference
- **Action**: Created HS-041

### 4. GKI android16-6.12 Latest Release
- Latest release: android16-6.12-2026-03 with kernel 6.12.69 (7 revisions, r1–r7)
- Previous branch: android16-6.12-2025-12 with kernel 6.12.58
- 16KB page size variants available on-demand
- No KMI detail changes beyond what HS-023 and HS-033 already capture

### 5. AOSP Biannual Release Cadence Confirmed
