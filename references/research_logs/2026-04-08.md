# Research Log — 2026-04-08

## Session Type
Periodic AOSP research session (steward agent)

## Topics Researched

1. **Android 16 GKI / Kernel 6.12 internals** — Deep dive into android16-6.12 changes beyond KMI baseline (HS-023). **NEW**: EEVDF scheduler replaces CFS, per-VMA locks for mmap_lock contention, proxy execution for priority inversion, RCU_LAZY, CONFIG_ZRAM_MULTI_COMP, Clang 19.0.1 stricter `__counted_by` bounds checking, CONFIG_OF_DYNAMIC exposing driver device-tree ref-counting bugs. Created HS-033.

2. **Generic Bootloader (GBL)** — **NEW**: Android 16 introduces a standardized, updatable bootloader (GBL). Major shift from vendor-specific LK/U-Boot. Designed for bootloader OTAs and boot flow standardization. Not yet visible in public AOSP tree. Created HS-034.

3. **Android 16 Multimedia** — **NEW**: CAP AIDL gap (documented in HS-025/HS-032) is now fixed in A16. APV codec support for professional video. Platform transitioning to AV1 from VP8/VP9/H.264. HDR enhancements (SDR fallback, HLG/DolbyVision capture). Media Quality Framework for Android TV. Created HS-035.

4. **Build System — additional changes** — **NEW**: Partition image isolation (only PRODUCT_PACKAGES contents), module name character restrictions, genrule directory inputs disallowed, M4 removed from PATH, ALLOW_NINJA_ENV becoming default, BOARD_HAL_STATIC_LIBRARIES deprecated. Bazel migration incremental. Created HS-036.

5. **Android 16 Security/SELinux** — KeyMint 4.0 attestation certificates with `moduleHash` field for APEX verification. Cellular security 2G toggle refinement. Private Space file import (QPR2). IOCTL hardening already captured in HS-027.

6. **Android 16 Connectivity** — Bluetooth AIS GATT service for OS identification. New IMS service APIs (EpsFallbackReason, ImsTrafficType, ImsTrafficDirection). Wi-Fi Hotspot disconnect callback. Ranging module aggregating UWB, BT channel sounding, RSSI, Wi-Fi RTT. Extends HS-029.

7. **Android 16 AVF/pKVM** — Additional detail beyond HS-024: Trusty OS for standard trusted apps in protected VMs, vendor-private SMC support, platform device assignment to pVMs, hypervisor structured logging/tracing.

8. **AOSP release model** — Confirmed: 2026 onward, AOSP source published in Q2 and Q4 only (trunk stable model). Use `android-latest-release` branch, not `aosp-main`.

9. **16KB page size** — No change from HS-028. Mandatory by May 31, 2026.

10. **Android 16 Framework Services** — Desktop windowing, heart rate sensor permission rename (SENSOR_PERMISSION_READ_HEART_RATE), PWLE haptics API with baseline primitives, seamless app updates (dexopt moved earlier in install).
